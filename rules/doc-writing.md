# 문서 작성 규칙 — loki-practice

## 기본 원칙
1. 모든 문서는 한국어로 작성하되, 기술 용어(LogQL, stream, label, ingester 등)는 영어 그대로 사용
2. 제목은 `# [주제] — Loki [문서유형]` 형식으로 통일
3. 문서 상단에 작성일, 작성자, 검토자 메타데이터 포함
4. 코드 블록에는 반드시 언어 지정 (```logql, ```yaml, ```bash)
5. 모든 LogQL 예시는 실행 가능한 형태로 제공

## 문서 구조
- **개요**: 목적과 수집 대상 (3-5문장)
- **레이블 스키마**: 사용 레이블 테이블
- **LogQL 예시**: 주요 쿼리 모음
- **운영**: 체크리스트 및 명령어
- **트러블슈팅**: 증상-원인-해결 테이블

## LogQL 문서화 규칙
- 쿼리에 주석으로 목적 설명
- 스트림 셀렉터 → 필터 → 파서 → 포맷 순으로 작성
- logcli 테스트 명령어 함께 제공
- 카디널리티 높은 레이블 사용 시 주의사항 명시

## 금지 사항
- 개인 식별 정보 포함 금지
- 프로덕션 자격증명 문서에 포함 금지
- 검증되지 않은 LogQL 쿼리 문서화 금지
